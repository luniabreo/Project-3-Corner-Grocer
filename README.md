The Corner Grocer project focused on building a program that analyzes a text file of grocery transactions and produces useful frequency data.
The goal was to help the store understand how often each item is purchased so they can make better inventory and stocking decisions. 
The program reads the input file, counts occurrences of each item, and outputs the results in multiple formats—including a full frequency list, a single‑item lookup, and a histogram. 
This solved the problem of manually tracking item popularity and provided a simple, automated way to generate insights from raw transaction data.

What I Did Particularly Well
I did a strong job structuring the program in a clean, modular way. My use of functions, maps, and clear separation between input handling, data processing, and output made the program easy to follow and maintain.
I also handled file input robustly and ensured the program produced consistent, readable results for the user.

Where I Could Enhance My Code
I could improve the program by adding stronger input validation, especially around user menu choices and item lookups.
I could also enhance efficiency by trimming whitespace, normalizing case, and handling unexpected formatting in the input file. 
These improvements would make the program more secure, more resilient to bad data, and more user‑friendly. Additionally, encapsulating the frequency logic into a dedicated class or struct would make the program more extensible for future features.

Most Challenging Parts and How I Overcame Them
The most challenging part was implementing the frequency map and ensuring the program handled all variations of user input correctly. 
I overcame this by breaking the problem into smaller steps, testing each function individually, and using debugging output to verify that the map was storing and updating values correctly.
I also relied on course resources, zyBooks examples, and documentation for C++ maps and file streams to guide my approach.
